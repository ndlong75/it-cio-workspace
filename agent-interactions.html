SNOW
→
Infra · Security · Dev · PM · CIO
AUTO 7:30
WEBHOOK P1
❄️ Snow Agent là "dispatcher" trung tâm — phân loại và route toàn bộ Snow queue
⏰ 07:30 Daily
(Auto)
→ Infra: Route incidents category=Infrastructure. Notify về maintenance windows, CI health alerts từ CMDB
Infra không cần tự scan Snow — Snow Agent đã lọc và push đúng việc
⏰ 07:30 Daily
(Auto)
→ Security: Route incidents category=Security. Push access requests cần review. Flag unassigned sec tickets
Security tập trung điều tra, không mất thời gian scan queue
⏰ 07:30 Daily
(Auto)
→ Dev: Route bug reports (defect category). Push enhancement requests liên quan code
Dev biết chính xác bug nào cần fix trong sprint
⏰ 07:30 Daily
(Auto)
→ PM: Route enhancement REQs từ phòng ban cần triage. Flag requests pending quá lâu
PM biết ngay có yêu cầu mới nào cần assess và estimate
⚡ Khi P1/P2
tạo mới
→ CIO: Alert P1 trong 15 phút. SLA breach warning. CHG cần CAB approval hôm nay
CIO không bị surprise, luôn biết critical issues trước khi chúng leo thang
⏰ 14:00 Daily
(Auto)
→ Tất cả: SLA breach warning — ticket của agent nào sắp breach → ping đúng agent đó
Cảnh báo 2h trước khi breach để có đủ thời gian xử lý
CIO
→
Snow · Infra · Security · Dev · PM
TELE DECISION
🔵 CIO ra quyết định cuối → agents thực thi
⚡ Sau Snow
CAB review
→ Snow: APPROVE / REJECT / DEFER CHG với reasoning. Snow Agent update Snow record và notify implementer
CIO là người duy nhất có quyền approve production changes
⚡ Khi P1 >30'
chưa resolve
→ Infra / Snow: Activate DR plan nếu cần. Authorize emergency resources. Escalate to vendors
P1 kéo dài cần CIO authorization để bypass normal processes
⚡ Khi nhận
Board request
→ PM: Approved request → PM tạo project charter, allocate resources. Rejected → PM notify requester
PM cần CIO decision trước khi commit team capacity
⚡ Monthly
Board Report
→ Dev: Tech debt sprint approved? Architecture decision? Library upgrade OK? CIO sign-off
Major technical decisions cần CIO alignment với business strategy
⚡ Quarterly
→ Infra: Capacity investment approved. Cloud budget OK. DR test schedule confirmed
Capex decisions cần CIO approval trước khi Infra act
INFRA
→
CIO · Snow · Security · Dev · PM
EVENT-DRIVEN
AUTO 07:00
🟠 Infra Agent là nền tảng — tất cả agents phụ thuộc vào hạ tầng stable
⏰ 07:00
Health Check
→ CIO: Morning infrastructure report: uptime đêm qua, backup status, server health summary
CIO cần infrastructure baseline trước khi đọc daily briefing 8:00
⚡ Khi P1
infrastructure
→ Snow: Tạo / update Snow INC với work notes, probable cause, fix actions. Close sau resolve + PIR
Snow là system of record — mọi incident phải được documented
⚡ Khi patch
xong hoặc CVE
→ Security: "Server X đã patch CVE-YYYY-NNNN." Security verify và close vulnerability tracking
Security cần confirm patch thực sự applied, không chỉ nói sẽ patch
⚡ Trước mỗi
deploy
→ Dev: "Staging env ready" / "Production maintenance window: 22:00-23:00 thứ 6" / "Rollback env confirmed"
Dev không thể deploy nếu không có Infra confirmation về environment readiness
⏰ Weekly
Friday
→ PM: Capacity forecast: "Infra có thể support thêm 2 dự án Q2 với current resources. Q3 cần thêm server"
PM cần infra capacity data để lên kế hoạch project pipeline

→
CIO · Infra · Dev · PM · Snow
ALERT-DRIVEN
AUTO 07:30
🔴 Security Agent cross-cuts tất cả — bảo mật là concern của toàn hệ thống
⚡ Khi phát
hiện breach
→ CIO: Immediate notification: "Data breach suspected. Customer PII potentially exposed. Regulatory report needed within 72h"
CIO phải activate crisis protocol và notify Board, Legal ngay lập tức
⚡ Khi CVE
critical mới
→ Infra: "CVE-YYYY-NNNN affects Oracle 19c on prod-db-01. CVSS 9.8. Patch required within 72h. Staging env first"
Infra cần CVE details để prioritize và execute patch với đúng urgency
⚡ Khi phát
hiện trong code
→ Dev: "Dependency log4j-2.14 has CRITICAL CVE. Block release until updated. Here's remediation guide"
Dev cần security alerts để block unsafe releases và fix vulnerable code
⏰ Thứ 6
17:00 Weekly
→ PM: "Nhân viên A nghỉ việc Thứ 4. Account đã disabled. Tất cả project access revoked. PM update stakeholder list"
PM cần biết team member changes để update project contacts và access
⚡ Khi sec
incident xảy ra
→ Snow: Update Snow Security INC: work notes, forensics findings, containment actions, closure sau eradicate
Security incidents phải có full audit trail trong Snow cho compliance
DEV
→
CIO · Snow · Infra · Security · PM
AUTO 09:00
POST-EVENT
🟣 Dev Agent báo cáo delivery và phối hợp deployment với Infra/Security
⚡ Khi sprint
at risk
→ CIO + PM: "Sprint 12 delay risk: dependency on vendor API delayed 1 tuần. Need decision: descope feature X or extend sprint?"
CIO và PM cần cùng quyết định scope vs timeline tradeoff
⚡ Sau mỗi
deployment
→ Snow: Update Snow CHG: implementation done, post-deployment check results, close CHG với PIR notes
CHG phải closed properly — audit trail cho mọi production change
⚡ Trước
deployment
→ Infra: "Release v2.3.1 ready to deploy. Requires DB migration. Estimated 45 min downtime. Need maintenance window confirmation"
Infra phải prepare environment, schedule maintenance, có rollback ready
⚡ Khi code
có sec issue
→ Security: "Found hardcoded credential in PR #234. Blocked merge. Need security review before proceeding"
Security cần review và confirm fix trước khi vulnerable code được merge
⏰ End of
sprint
→ PM: Sprint velocity report: "Completed 42/50 story points. 2 stories carry-over. Bug fix took 30% more than estimated"
PM cần accurate velocity data để update project timeline và stakeholder forecast
PM
→
CIO · Dev · Infra · Security · Snow
AUTO Thứ 2
COORDINATION
🟢 PM Agent là coordinator — đảm bảo tất cả agents aligned với project goals
⚡ Khi risk
escalate
→ CIO: "Project X risk: vendor delay sẽ cause 3-week slip. Options: (A) extend timeline, (B) hire contractor, (C) descope. Need decision by EOD"
CIO cần context đầy đủ để quyết định, PM prepare options analysis
⏰ Đầu sprint
(bi-weekly)
→ Dev: Sprint planning input: "Business priority này tuần: Feature A > Bug B > Enhancement C. Stakeholder demo Thứ 6"
Dev cần business priority context để order sprint backlog đúng
⚡ Khi project
cần infra
→ Infra: "Project Y cần thêm 2TB storage và 1 server từ Q2. Budget approved. Lead time: cần trước 15/4"
Infra cần advance notice để order hardware đúng lead time
⚡ Khi staff
thay đổi
→ Security: "Nhân viên B chuyển từ IT sang Finance từ Thứ 2. Cần revoke IT system access, grant Finance access"
Security cần thay đổi access kịp thời khi role thay đổi
⚡ Sau triage
enhancement
→ Snow: "Enhancement REQ-12345 assessed: 3-week effort, approved for Q3 sprint. Update Snow REQ status và notify requester"
Snow REQ phải được updated với timeline để requester biết khi nào sẽ được delivery
